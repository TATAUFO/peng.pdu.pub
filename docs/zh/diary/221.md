# 221. PDU的再次简化

关于PDU的再次简化。

身份之间的群组关系不应有某身份发布特别的信息构建，而是通过信息之间的公开交互所构建。无论使用者喜欢或者不喜欢一条信息，都看见了这个信息，而且互动的行为本身就表明了信息的意义。

这种身份关联构建方式更加回归视界这个词的本意，看到了这条信息，信息便对你产生影响，这才是信息的意义，而用户喜欢的人或者信息。真实世界并不是光有喜欢的个体所构成，而是有所有能所见的个体构成。

在这种想法的支撑下，每个用户或者使用者的世界只有一个。但是他看到的世界跟他有互动的世界是不同的，别人看到的是与他互动的世界，而他看到的可能更多，这样等同于由用户的行为来过滤。

> 或者再明确一些，看到的是别人的世界的组合，互动的是自己的世界。

发布消息和引用或回复消息是单条消息，其引用只有一个，自引用除外。

而点赞可能是个异步的发布消息的行为，其一条消息可以引用多个被点赞的内容，以减小消息总量。

> 这需要更改白皮书。
