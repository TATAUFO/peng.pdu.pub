# 181. PDU组织形式的调整想法

当Community「社区」的概念被Species「物种」取代的，Invite「邀请」被Identity「认定」取代之后，PDU的组织方式随之也会有一些改变，创建的消息会同化为认定类型的消息，相当于任何使用者都可以表达对于其他的个体在某方面的认同，这个方面即Species，就是某个物种。

> 假设A同意B的观点。用PDU里面的概念加以说明，其含义如下。A作为某种自我身份，对于B次观点背后的那个自我身份表示两者的自我身份类似。这并不妨碍A会具备其他的自我身份，也不表达A对于B所有自我身份的认同。

每个使用者都能够进行类似的认定。具体的规则中，邀请的signatureDefine是一个原始的认定，则表示那个认定的扩展，而如果是其他，比如文字等，则表示一个新的认定。

对于认定，应取消Co-Sign「联署」下限和Identify「认定」上限的数量限制，将判定的方式给遇到信息的获取者。每个信息的获取者都应该有自己可见的所有身份信息消息列表，或至少是消息统计数据，以便根据信息或统计数据对于自身可见的信息获取范围进行调整。
