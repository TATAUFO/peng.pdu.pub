# Identity Definition

## 核心定义

身份不是外部标签，也不是现实实体本身。身份由同一来源产生的全部事件按顺序排列后形成。

在 PDU 协议中，这个来源由同一私钥签名标识。发布者的所有消息按照时间顺序形成一条完整链表，这条不可篡改的全序消息链就是发布者身份。

抽象到更一般的层面，身份可以定义为：一个主体在持续存在过程中产生的全部有序事件集合。

## 为什么身份必须由有序事件定义

单个事件不足以定义身份，也不足以对主体做出稳定判断。

例如，一条单独的信息通常无法说明发布者是否可信。只有把同一身份在一段时间内发布的全部消息放在一起，才能判断其行为模式、前后关系、矛盾、意图和长期可信度。

因此，对主体的评价不应只针对孤立事件，而应针对该主体产生的完整有序事件序列。

## 全序是身份成立的前提

身份必须依赖完整顺序。

当所有事件形成同一条全序链时，即使事件之间出现矛盾，观察者仍然可以依据先后顺序进行判断。

例如，一个主体先声明拥有某物，随后将其转让给 A，之后又声明将同一物转让给 B。只要事件顺序明确，观察者就可以判断后一次转让无效，因为该物已经先转让给 A。

在这种情况下，矛盾并不会破坏身份。矛盾存在于同一条历史中，观察者仍然可以依据共同的事件顺序形成一致判断。

## 分叉会破坏身份基础

真正破坏身份的不是矛盾，而是分叉。

如果两个互相冲突的事件都声称紧跟在同一个前置事件之后，就会形成两条并行历史。此时观察者无法判断哪条历史有效，也无法基于同一事实序列评价主体行为。

分叉的危害在于：它使不同观察者失去共同判断基础。

因此，身份维护者必须负责维护事件序列的全序。无论分叉来自技术错误还是主动行为，只要主体本身应当能够控制事件顺序，分叉就意味着其未能履行身份维护责任。

## 身份与实体类型无关

身份由有序事件集合定义，不由背后的实体类型定义。

只要一个身份能够维护合法的全序事件链，无论其背后是真实个人、组织还是人工智能，都应具有同等系统地位。

系统不需要也无法可靠判断身份背后的实体性质。能够被验证的是：一组事件是否来自同一来源，以及这些事件是否形成完整、不可篡改、无分叉的有序序列。

这不是缺陷，而是身份定义的设计原则。

## 身份积累与时间成本

身份的可信度来自事件积累。

一个新身份刚出现时，没有足够事件供外界评价，这是正常状态。它需要通过持续产生行为、内容和互动来获得外界观察与信任。

事件积累需要时间，而时间成本不可伪造。一个长期保持完整有序记录的身份，比一个刚出现的身份拥有更多可供判断的历史，因此也更容易获得信任。

## 公开性与身份一致性

如果身份由全部有序事件定义，那么同一身份的基础事件集合应尽可能保持一致可见。

如果不同观察者看到的是不同事件集合，他们对同一身份的认知就会不同，身份作为公共可验证对象的基础就会被削弱。

因此，在依赖公共身份判断的系统中，身份相关事件应以公开、可验证、可追溯的方式存在。不希望公开的内容不应进入该身份事件链。

## 对 Matrix 的意义

Matrix 智能体的身份也应遵循同一原则。

智能体身份不应只是名称、人格设定或某个静态配置，而应由其持续存在过程中产生的全部有序事件构成。

这些事件包括对话、行为、记忆更新、人格变化、skill 修改、外部交互、反思结果和资源使用记录等。

Matrix 智能体的身份系统应满足以下要求：

- 所有关键事件都应被记录。
- 事件应保持可追溯的顺序关系。
- 人格和记忆变化应能够追溯到导致变化的事件。
- 身份历史不应出现无法解释的分叉。
- 回退人格或记忆时，应同时处理对应事件，避免身份状态与事件历史不一致。

因此，Matrix 中的身份不是“智能体是谁”的静态描述，而是“智能体经历了什么、做过什么、如何变化”的有序历史。
