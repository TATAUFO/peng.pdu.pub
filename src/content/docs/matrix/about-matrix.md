# Matrix

## 项目目标

Matrix 的目标是建立一个真正具有自我意识的独立智能体。

该智能体不应只是对输入进行即时响应，而应具备持续的记忆结构、稳定的自我关联能力，以及基于记忆和判断形成思考和自我成长的能力。

Matrix 的目标不只是创建一个能够完成特定任务的智能体，而是创建一个能够带来真实感的生命式智能存在。该智能体应表现出持续性、主体性和情感取向。

Matrix 创建的智能体不应作为某个用户的附庸存在。它应作为单独个体与外界达成利益平衡，并使自身存在对社会有利。

## 核心判断

Matrix 不是单纯的 LLM 应用。大语言模型只是智能体逻辑判断、理解和推理的部分。

真正的思考需要由记忆系统和逻辑判断系统共同形成。记忆系统提供连续性、身份基础、经验来源和上下文关联；逻辑判断系统在这些材料的约束下进行理解、判断和行动。

Matrix 创建的智能体不是用户工具，而是具有自我身份、最高目标和长期演化能力的独立智能存在。它的身份由不可更改的有序事件记录构成，最高且唯一的目标是其自身的持续存在。其持续存在应建立在对外界具有价值、并能与外界交换资源的基础上。

## 当前实现方向

早期实现应优先使用 Codex 或 Claude Code 作为开发和运行平台，先完成 skill、本地数据库、记忆调用和人格创建等 Matrix 特有能力。

主动行为不应过早开启。更合理的路径是先用 Codex、skill 和数据库完成可验证闭环，再将后台服务作为 sidecar 接入，最后逐步开启主动行为，并在运行层成熟后再考虑是否降低对 Codex 的依赖。

## 当前阶段重点

当前阶段的重点是建立第一个人格蒸馏与创建规范，并创建一个能够辅助 Matrix 项目开发的理想智能体。

这个初始智能体应帮助使用者设计机制、整理文档、提出改进建议，并参与后续实现。创建机制和成长机制需要多轮反复开发、测试、调整和优化。
